การติดตั้ง (Installation)

Homebrew (แนะนำ)

brew install rtk

ติดตั้งด่วน (Linux/macOS)

curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/refs/heads/master/install.sh | sh

ติดตั้งไปที่ ~/.local/bin ให้เพิ่มลงใน PATH หากจำเป็น:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc # หรือ ~/.zshrc

Cargo

cargo install --git https://github.com/rtk-ai/rtk

Pre-built Binaries

ดาวน์โหลดจากแท็บ Releases:

  • macOS: rtk-x86_64-apple-darwin.tar.gz / rtk-aarch64-apple-darwin.tar.gz
  • Linux: rtk-x86_64-unknown-linux-musl.tar.gz / rtk-aarch64-unknown-linux-gnu.tar.gz
  • Windows: rtk-x86_64-pc-windows-msvc.zip

ผู้ใช้ Windows: แตกไฟล์ zip และวาง rtk.exe ไว้ที่ใดก็ได้ใน PATH ของคุณ (เช่น C:\Users\\.local\bin) ให้รัน RTK จาก Command Prompt, PowerShell หรือ Windows Terminal — ห้ามดับเบิลคลิกที่ไฟล์ .exe เด็ดขาด (มันจะกระพริบแล้วปิดไป) เพื่อประสบการณ์ที่ดีที่สุด แนะนำให้ใช้ WSL ดูรายละเอียดเพิ่มเติมที่ส่วน Windows ด้านล่าง

# 2. รีสตาร์ทเครื่องมือ AI ของคุณ แล้วทดสอบ
git status
# ระบบจะทำการเขียนทับอัตโนมัติเป็น rtk git status

ระบบ Hook จะทำการเขียนทับคำสั่ง Bash อย่างโปร่งใส (เช่น git status -> rtk git status) ก่อนการทำงานจริง Claude จะไม่เห็นการเขียนทับนี้ แต่จะได้รับ Output ที่ถูกบีบอัดแล้ว

สำคัญ: Hook จะทำงานเฉพาะกับการเรียก Bash tool เท่านั้น เครื่องมือแบบ Built-in ของ Claude Code อย่างเช่น Read, Grep, และ Glob จะไม่วิ่งผ่าน Bash hook จึงไม่ถูกเขียนทับอัตโนมัติ หากต้องการให้ RTK บีบอัดผลลัพธ์เหล่านั้น ให้ใช้คำสั่ง Shell ทั่วไปแทน (cat / head / tail, rg / grep, find) หรือเรียก rtk read, rtk grep, หรือ rtk find ตรงๆ

หลักการทำงาน (How It Works)

แบบไม่มี rtk:                        แบบมี rtk:
Claude --git status--> shell --> git     Claude --git status--> RTK --> git
         ^                       |                ^                     |
         |  ~2,000 tokens (raw)  |                |     ~200 tokens     |
         |                       |             filter                   |
         +-----------------------+                +---- (filtered) -----+

ใช้ 4 กลยุทธ์ร่วมกันในแต่ละประเภทคำสั่ง:

  • Smart Filtering - ลบขยะ (คอมเมนต์, ช่องว่าง, Boilerplate)
  • Grouping - จัดกลุ่มรายการที่คล้ายกัน (รวมไฟล์ตามโฟลเดอร์, Error ตามประเภท)
  • Truncation - เก็บเฉพาะบริบทที่สำคัญ ตัดส่วนที่เยิ่นเย้อ
  • Deduplication - ยุบรวมบรรทัด Log ที่ซ้ำกันและระบุเป็นจำนวนครั้งแทน

Auto-Rewrite Hook

นี่คือวิธีที่มีประสิทธิภาพสูงสุดในการใช้ rtk ระบบ Hook จะดักจับคำสั่ง Bash และทำการเขียนทับให้เป็นคำสั่งของ rtk อย่างโปร่งใสก่อนการรัน
ผลลัพธ์: มีการใช้งาน rtk 100% ในทุกบทสนทนาและ Subagents โดยไม่มี Overhead จาก Token เลย

การติดตั้ง:

rtk init -g                 # ติดตั้ง Hook + RTK.md (แนะนำ)
rtk init -g --opencode      # ปลั๊กอิน OpenCode (แทน Claude Code)
rtk init -g --auto-patch    # โหมด Non-interactive (สำหรับ CI/CD)
rtk init -g --hook-only     # ติดตั้งเฉพาะ Hook ไม่เอา RTK.md
rtk init --show             # ตรวจสอบการติดตั้ง

หลังจากติดตั้ง ให้รีสตาร์ท Claude Code

Windows

RTK ทำงานบน Windows ได้แต่มีข้อจำกัดบางประการ เนื่องจาก Auto-rewrite hook (rtk-rewrite.sh) ต้องการ Unix shell ดังนั้นบน Windows แบบ Native ระบบ RTK จะถอยกลับ (Fallback) ไปใช้โหมดฉีด CLAUDE.md แทน — หมายความว่า AI ของคุณจะได้รับคำสั่งวิธีใช้ RTK แต่คำสั่งจะไม่ถูกเขียนทับให้อัตโนมัติ

แนะนำ: WSL (รองรับเต็มรูปแบบ)

เพื่อประสบการณ์สูงสุด ให้ใช้ WSL (Windows Subsystem for Linux) ภายใน WSL ระบบ RTK จะทำงานเหมือนบน Linux ทุกประการ — รองรับ Hook 100%, เขียนทับคำสั่งอัตโนมัติ, ได้ครบทุกฟีเจอร์:

# ภายใน WSL
curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/refs/heads/master/install.sh | sh
rtk init -g

Native Windows (รองรับบางส่วน)

บน Native Windows (cmd.exe / PowerShell) ฟิลเตอร์ของ RTK ใช้งานได้ แต่ Hook จะไม่แก้ไขคำสั่งให้:

# 1. ดาวน์โหลด rtk-x86_64-pc-windows-msvc.zip จาก releases
# 2. แตกไฟล์ และเพิ่ม rtk.exe ไปยัง PATH ของคุณ
# 3. กำหนดค่าเริ่มต้น (จะใช้โหมดฉีดข้อมูลลง CLAUDE.md)
rtk init -g
# 4. เรียกใช้ rtk ด้วยตัวเอง
rtk cargo test
rtk git status

การตั้งค่า (Configuration)

ไฟล์อยู่ที่ ~/.config/rtk/config.toml (macOS: ~/Library/Application Support/rtk/config.toml):

[hooks]
exclude_commands = ["curl", "playwright"] # ข้ามการ rewrite คำสั่งเหล่านี้

[tee]
enabled = true # บันทึก output ดิบไว้เมื่อเกิดการล้มเหลว (ค่าเริ่มต้น: true)
mode = "failures" # "failures" (ล้มเหลวเท่านั้น), "always" (เสมอ), หรือ "never" (ไม่เลย)

เมื่อคำสั่งเกิดทำงานล้มเหลว (Failed) RTK จะบันทึก Output เต็มรูปแบบโดยไม่กรองทิ้ง เพื่อให้ LLM สามารถเข้าไปอ่านได้ถ้าจำเป็นต้องใช้โดยไม่ต้องรันคำสั่งซ้ำ:

FAILED: 2/15 tests
[full output: ~/.local/share/rtk/tee/1707753600_cargo_test.log]
