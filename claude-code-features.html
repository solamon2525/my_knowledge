Claude.md
ไฟล์หน่วยความจำโปรเจกต์
01

ไฟล์ Markdown ที่ Claude อ่านอัตโนมัติทุกครั้งที่เริ่มเซสชัน ทำให้ Claude "จำ" บริบทโปรเจกต์ได้โดยไม่ต้องอธิบายซ้ำ

  • บันทึก Tech Stack, Framework, ข้อกำหนดโปรเจกต์
  • กำหนด Coding Style และ Naming Convention
  • ระบุคำสั่ง build, run, test ที่ใช้บ่อย
  • ตั้งกฎห้าม เช่น "อย่าแก้ไขไฟล์ config โดยไม่ถาม"
  • วางไว้ใน root หรือ nested หลายโฟลเดอร์ได้
💡 เคล็ดลับ สร้าง CLAUDE.md ใน ~/.claude/ สำหรับกฎ global ที่ใช้ทุกโปรเจกต์ เช่น "ตอบเป็นภาษาไทยเสมอ"
~/.claude/CLAUDE.md → global rules
./CLAUDE.md → project-specific rules
หน่วยความจำถาวร ไม่ต้องอธิบายซ้ำ
🔐
Permissions
ระบบจัดการสิทธิ์การเข้าถึง
02

ควบคุมได้ละเอียดว่า Claude จะใช้เครื่องมือใดได้บ้าง ทั้งแบบอนุญาตถาวร อนุญาตครั้งเดียว และห้ามใช้โดยเด็ดขาด

  • Allow — อนุญาตถาวร ไม่ถามซ้ำ (เช่น อ่านไฟล์)
  • Ask — ถามทุกครั้งก่อนดำเนินการ
  • Deny — ห้ามใช้เครื่องมือนั้นโดยสิ้นเชิง
  • กำหนดได้ทั้ง per-session และ per-project
  • ล็อกไม่ให้ลบไฟล์สำคัญโดยอัตโนมัติ
⚠️ ข้อควรระวัง ใน production ตั้ง bash เป็น "Ask" เสมอ เพื่อป้องกัน Claude รันคำสั่งอันตรายโดยไม่ตั้งใจ
"permissions": {
"bash": "ask",
"read": "allow",
"write": "ask"
}
ความปลอดภัย ควบคุมละเอียด
📋
Plan Mode
โหมดวางแผนก่อนลงมือทำ
03

Claude สร้างแผนการทำงานให้ดูก่อน โดยไม่แตะไฟล์หรือรันโค้ดใดๆ จนกว่าคุณจะอนุมัติ ลดความผิดพลาดได้มาก

  • วิเคราะห์ปัญหาและเสนอแนวทางแก้ไขหลายแบบ
  • แสดงรายการไฟล์ที่จะถูกสร้าง/แก้ไข/ลบ
  • ช่วยจับข้อผิดพลาดในแผนก่อนที่จะสายเกินไป
  • เหมาะสำหรับงาน refactor ขนาดใหญ่ หรืองาน critical
  • ใช้ร่วมกับ Checkpoints เพื่อความปลอดภัยสูงสุด
🎯 วิธีใช้งาน กด Shift+Tab หรือพิมพ์ "วางแผนก่อน อย่าลงมือทำ" เพื่อเข้าสู่ Plan Mode ได้ทันที
Shift+Tab → เปิด Plan Mode
"plan only, don't execute yet"
ป้องกันความผิดพลาด ตรวจก่อนทำ
⏱️
Checkpoints
จุดบันทึกสถานะอัตโนมัติ
04

Claude สร้าง snapshot ของโค้ดอัตโนมัติก่อนทุกการเปลี่ยนแปลง ย้อนกลับได้ทันทีหากผลลัพธ์ไม่เป็นที่พอใจ

  • บันทึกอัตโนมัติก่อนทุกครั้งที่แก้ไขไฟล์
  • ย้อนกลับไปจุดใดก็ได้ในประวัติการทำงาน
  • ดูความแตกต่างระหว่าง checkpoint (diff view)
  • ทดลองแนวทางหลายแบบโดยไม่กลัวพัง
  • ทำงานคู่กับ git เพื่อความปลอดภัยเป็นชั้นๆ
💡 เคล็ดลับ บอก Claude ว่า "สร้าง checkpoint ก่อน" ก่อนทำงาน risky เช่น refactor ครั้งใหญ่หรือลบ module
v1 (เริ่มต้น) → v2 (ฟีเจอร์ A)
v3 ← อยู่ตรงนี้
↩ ย้อนกลับ v2 ได้ทันที
ย้อนเวลาได้ ทดลองอย่างปลอดภัย
🧩
Skills
ชุดคำสั่งนำกลับมาใช้ซ้ำได้
05

บันทึกวิธีทำงานที่ซับซ้อนไว้เป็น Skill แล้ว Claude จะอ่านและปฏิบัติตามโดยอัตโนมัติทุกครั้งที่งานนั้นเกิดขึ้น

  • สร้าง Skill สำหรับงานที่ทำซ้ำบ่อย เช่น "สร้าง component"
  • เก็บ best practice ของทีมไว้ใน .claude/skills/
  • Skill สามารถ import Skill อื่นได้ (nested skills)
  • แชร์ Skill ข้ามโปรเจกต์ในทีมผ่าน git ได้
  • ระบุเงื่อนไขเรียกใช้ด้วย keyword หรือ pattern
📂 ตัวอย่าง Skill สร้าง skill "สร้าง API endpoint" ที่ระบุ pattern ของ controller, route, validation และ test ไว้ในที่เดียว
.claude/skills/
create-component.md
deploy-firebase.md
write-thai-test.md
นำกลับมาใช้ได้ มาตรฐานทีม
🪝
Hooks
ทริกเกอร์การทำงานอัตโนมัติ
06

กำหนดสคริปต์ที่รันอัตโนมัติในช่วงเวลาสำคัญของ workflow เหมือน Git Hooks แต่ทรงพลังและยืดหยุ่นกว่ามาก

  • PreToolUse — รันก่อน Claude ใช้เครื่องมือ (ตรวจสอบ, log)
  • PostToolUse — รันหลังจากใช้เครื่องมือ (format, lint)
  • Notification — รับแจ้งเตือนเมื่อ Claude ทำงานเสร็จ
  • Stop — ทำงานเมื่อ session จบลง (cleanup, report)
  • รัน shell script, Python, Node.js ได้ทุกอย่าง
🔥 ใช้งานจริง ตั้ง PostToolUse hook รัน prettier --write อัตโนมัติหลังทุกครั้งที่ Claude แก้ไขไฟล์ JS
"hooks": {
"PostToolUse": [
"npm run lint --fix"
]
}
อัตโนมัติ 100% Workflow ทรงพลัง
🔌
MCP
เชื่อมต่อระบบและบริการภายนอก
07

Model Context Protocol มาตรฐานเปิดสำหรับเชื่อม Claude เข้ากับเครื่องมือและข้อมูลภายนอกได้ไม่จำกัด

  • ฐานข้อมูล: PostgreSQL, Firestore, Supabase, MySQL
  • บริการ Cloud: GitHub, Vercel, Cloudflare, AWS
  • เครื่องมือทีม: Slack, Gmail, Google Drive, Notion
  • สร้าง MCP server เองด้วย Python หรือ TypeScript
  • รัน MCP server ได้ทั้งแบบ local และ remote URL
🚀 กรณีใช้งานจริง ติดตั้ง firebase-mcp แล้ว Claude อ่านเขียน Firestore ได้โดยตรง ไม่ต้องเขียนโค้ดเชื่อมเองเลย
claude mcp add firebase \
--transport http \
--url http://localhost:3000
เชื่อมทุกระบบ มาตรฐานเปิด
🔧
Plugins
ขยายความสามารถแบบไม่ต้องโค้ด
08

เพิ่มความสามารถให้ Claude ด้วย integration สำเร็จรูป รองรับเครื่องมือยอดนิยมในวงการพัฒนาซอฟต์แวร์

  • Docker — รัน container และจัดการ image
  • pytest / Jest — รัน test suite และรายงานผล
  • ESLint / Prettier — ตรวจและจัดรูปแบบโค้ด
  • Playwright — ควบคุม browser อัตโนมัติ
  • เพิ่ม custom plugin ด้วยการนิยาม tool schema เอง
💡 เคล็ดลับ ใช้ Playwright plugin ให้ Claude ทดสอบ UI บน Chrome จริงๆ แทนการเดาว่าหน้าตาจะเป็นยังไง
Plugin ยอดนิยม:
Docker · Playwright · pytest
ESLint · GitHub Actions
ไม่ต้องโค้ดเพิ่ม เครื่องมือครบ
🎯
Context Window
ควบคุมสิ่งที่ Claude มองเห็น
09

จัดการ Context อย่างชาญฉลาด ส่งเฉพาะข้อมูลที่จำเป็นให้ Claude เพื่อประหยัด token และได้คำตอบที่แม่นยำขึ้น

  • เลือกไฟล์ที่ต้องการให้ Claude อ่านด้วย #ชื่อไฟล์
  • ล้าง context เพื่อเริ่มต้นใหม่เมื่อพูดถึงงานอื่น
  • กำหนด include/exclude ไฟล์ประเภทใดก็ได้
  • ดู context ที่ Claude กำลังใช้แบบ real-time
  • ตั้งค่า gitignore-style เพื่อกรองไฟล์ที่ไม่ต้องการ
⚡ เพิ่มประสิทธิภาพ ระบุไฟล์ที่เกี่ยวข้องโดยตรงแทนการให้ Claude ค้นหาเอง ประหยัด token และได้คำตอบเร็วกว่า
"แก้ไข #src/auth.js ให้รองรับ
JWT token แบบ refresh token"
ประหยัด Token คำตอบแม่นยำ
⚡
Slash Commands
คำสั่งลัดประหยัดเวลา
10

บันทึก prompt ที่ใช้บ่อยเป็นคำสั่งลัด เรียกใช้ได้ทันทีด้วยการพิมพ์ /ชื่อคำสั่ง ไม่ต้องพิมพ์ prompt ยาวซ้ำอีกต่อไป

  • Built-in: /clear · /compact · /review · /help
  • สร้าง custom command โดยบันทึกไฟล์ .md ใน .claude/commands/
  • รองรับ parameter เช่น /deploy staging หรือ /deploy prod
  • Command ใช้ได้ทุก project หรือเฉพาะ project ก็ได้
  • แชร์ command ข้ามทีมผ่าน git repository
🎯 คำสั่งที่นิยมใช้ /review — code review ทั้ง branch · /test — รัน test suite · /deploy — push และ deploy อัตโนมัติ
.claude/commands/
review.md → /review
deploy.md → /deploy
doc.md → /doc
รวดเร็ว ทำซ้ำได้ทุกเวลา
🗜️
Compaction
บีบอัดบทสนทนายาว
11

เมื่อบทสนทนายาวเกินไปจน context เกือบเต็ม Claude จะสรุปประวัติให้กระชับโดยอัตโนมัติ โดยไม่สูญเสียสาระสำคัญ

  • ทำงานอัตโนมัติเมื่อ context ใกล้เต็ม (200k+ tokens)
  • เรียกใช้เองด้วย /compact ได้ทุกเมื่อที่ต้องการ
  • Claude สรุป แต่เก็บโค้ดสำคัญและ decision ไว้
  • ประหยัดค่าใช้จ่าย API ได้ถึง 70% ในงานระยะยาว
  • ดีกว่าการเริ่ม session ใหม่ เพราะยังจำ context ได้
💰 ประหยัดเงิน รัน /compact ก่อนเริ่มงานใหม่ในวันถัดไป แทนการเปิด session ใหม่ที่ต้องอธิบาย context ซ้ำทั้งหมด
/compact → สรุป context ทันที
ประหยัด token สูงสุด 70%
ยังจำบริบทเดิมได้ทั้งหมด
ประหยัด Token ไม่สูญเสียบริบท
🤖
Subagents
ทีม AI ทำงานคู่ขนานกัน
12

ให้ Claude สร้างทีม AI ขนาดเล็กทำงานพร้อมกันแบบ parallel เหมาะสำหรับงานซับซ้อนที่แบ่งออกเป็นส่วนๆ ได้

  • Lead Agent แบ่งงานและส่งให้ Subagent แต่ละตัว
  • Subagent ทำงานพร้อมกัน (ไม่รอคิว) ประหยัดเวลามาก
  • แต่ละ agent มี context, tools และ scope ของตัวเอง
  • รวบรวมผลลัพธ์และ merge เข้าด้วยกันอัตโนมัติ
  • ใช้ได้กับ: test, build, scrape, analyze พร้อมกัน
🚀 กรณีใช้งาน Agent 1 เขียนโค้ด · Agent 2 เขียน test · Agent 3 เขียน docs — ทำพร้อมกัน เสร็จใน 1/3 ของเวลา
Lead Agent
├── Agent A: เขียนโค้ด
├── Agent B: เขียน test
└── Agent C: เขียน docs
ทำงานพร้อมกัน อนาคต AI
